# UX Research Challenge — Robinhood Cash-Out Redesign

Single-page UX case study for the **SnapRefund UI/UX Technical Challenge**: a research-backed redesign of **Robinhood’s withdrawal (cash-out) flow**, with competitive analysis, pain points, annotated real-app screenshots, and an interactive before/after prototype.

**Author:** Dhairya Ameria

## How to view

1. Clone or download this folder.
2. Open **`index.html`** in Chrome, Safari, or Firefox (double-click or *File → Open*).
3. Use the top nav to jump to sections, or scroll. The **Interactive Prototype** section has prev/next controls for the current vs. redesigned phone flows.

> **Note:** Keep the JPG files next to `index.html` so image paths resolve correctly.

---

## Case study outline (high level)

1. **Background** — Why the withdrawal experience matters for Robinhood users.  
2. **Research** — Competitive comparison (Cash App, Wise, Venmo) and insights.  
3. **Problem definition** — Current flow map, annotated screenshots, six pain points (e.g. Transfers hidden, ETA late, no quick-amount chips).  
4. **Redesigned flow** — Fewer steps, clearer hierarchy, ETA and decisions surfaced earlier.  
5. **Interactive prototype** — Side-by-side current vs. redesign; optional embedded tutorial video.  
6. **Design decisions** — Rationale tied to pain points.  
7. **Validation plan** — What to test and how to measure success.  
8. **Conclusion** — Principles for trustworthy fintech UX.
